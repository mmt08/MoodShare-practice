# Документация по проектной (учебной) практике

## Цель практики

Целью практики являлось освоение системы контроля версий Git, платформы GitHub, языка разметки Markdown, а также разработка статического веб-сайта проекта.

## Выполнение практики

### Этап 1. Создание структуры репозитория

Была создана структура репозитория в соответствии с требованиями задания.

Созданы следующие каталоги:

* docs
* reports
* site
* src
* task

В репозитории также были созданы служебные файлы:

* README.md
* .gitignore

### Этап 2. Настройка Git

Был установлен Git для операционной системы Windows.

В процессе работы были изучены основные команды:

```bash
git init
git add .
git commit -m "message"
git push
git status
```

Также был создан удалённый репозиторий на платформе GitHub и выполнена его синхронизация с локальным репозиторием.

### Этап 3. Работа с Markdown

Был изучен синтаксис языка Markdown.

В процессе практики были подготовлены следующие документы:

* README.md
* task/README.md
* docs/practice_documentation.md

Для оформления документов использовались:

* заголовки;
* списки;
* таблицы;
* выделение текста;
* ссылки.

### Этап 4. Создание статического сайта

Для представления проекта MoodShare был разработан статический веб-сайт.

На сайте планируется размещение:

* информации о проекте;
* сведений об участниках;
* журнала разработки;
* полезных ресурсов.

При создании сайта используются технологии HTML5 и CSS3.

## Используемые инструменты

* Git
* GitHub
* Markdown
* HTML5
* CSS3
* Visual Studio Code

## Результат

В результате выполнения практики был создан GitHub-репозиторий, подготовлена документация в формате Markdown и начата разработка статического сайта проекта MoodShare.
